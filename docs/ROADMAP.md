## 優先度別タスクリスト

### 高優先度（必須）

1. **PR #47のマージ** - Phase 12-3完了
2. **基本動作確認** - 全シーン遷移テスト
3. **クリティカルバグ修正** - ゲーム進行を妨げるバグ
4. **データ整合性確認** - セーブ/ロード動作確認

### 中優先度（推奨）

1. **UIポリッシュ** - 見た目の統一感
2. **戦闘バランス調整** - 計略コスト、AI強度
3. **サウンド実装** - 最低限のBGM/SE
4. **追加シナリオ** - 2-3個

### 低優先度（あれば良い）

1. **ボイス実装**
2. **マルチプラットフォーム**
3. **実績システム**
4. **オンライン対戦**

## 技術的負債

### 要リファクタリング

1. **GameManager肥大化**
   - 責務を分離（SceneManager, DataManager等）
   - 依存性注入の検討

2. **イベント管理**
   - EventBusのイベント整理
   - メモリリーク対策（購読解除の徹底）

3. **UI階層**
   - プレハブ構造の統一
   - 共通コンポーネントの抽出

### 改善検討事項

1. **アセンブリ定義の見直し**
   - 循環参照の解消
   - コンパイル時間の最適化

2. **ScriptableObjectの管理**
   - アドレッサブル対応
   - ホットリロード対応

3. **テスト容易性**
   - インターフェース導入
   - モック可能な設計
